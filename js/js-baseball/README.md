### Feature

1. [x] 컴퓨터가 랜덤 숫자 3개를 생성한다.
2. [x] 유저가 랜덤 숫자 3개를 입력한다.
3. [x] 유저 숫자와 컴퓨터의 숫자를 비교하여 스트라이크, 볼, 낫싱을 판단한다.
4. [x] 3번 리턴값을 통해 스트라이크, 볼 개수를 센다.
5. [x] 4번 리턴 값을 통해 승리 혹은 패배를 판단한다.
6. [x] 패배했다면 숫자 입력을 다시 받는다.
7. [x] 승리했다면 게임을 다시 시작할 지 입력을 받고 다시 시작한다면 게임을 1번부터 다시 시작한다.

### Think

1. 컴퓨터 숫자는 일급 컬렉션을 이용. 정답을 보관하고 비교하는 로직을 위해 클래스를 따로 만들어 관리한다.
2. 랜덤 숫자는 테스트가 어렵기 때문에 랜덤 숫자와 값을 비교하는 로직은 테스트를 실시하지 않는다.
3. 어떤 값을 인스턴스 변수로 두고 어떤 값을 일반 변수로 둬야 할지를 고민하기
4. 랜덤 숫자 유닛 테스트를 쉽게 할 수 있는 방법 고민하기
